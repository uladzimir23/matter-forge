# ADR-004 Tailwind stays on CDN

## Контекст

В `index.html` инлайн-скрипт:
```html
<script src="https://cdn.tailwindcss.com"></script>
<script>
  tailwind.config = { /* inline конфиг */ }
</script>
```

При этом в `package.json` уже установлен `@tailwindcss/vite` и в
`vite.config.ts` он подключён как плагин. То есть у проекта два параллельных
Tailwind'а. CDN-скрипт — первый, плагин запасной.

## Проблема

CDN-Tailwind:
- **медленнее** (внешний запрос при загрузке страницы)
- **hardcoded JIT в рантайме** (все классы парсятся на клиенте)
- **зависит от CDN uptime**
- **нет purge** — все классы Tailwind доступны, в т.ч. те что не
  использованы

Локальный билд:
- быстрее
- purge работает, размер стилей меньше
- нет внешних зависимостей
- но требует переписать `tailwind.config` в standalone-файл и добавить
  `index.css` с `@tailwind base; @tailwind components; @tailwind utilities;`

## Решение

**Пока оставить CDN.** Миграция отложена в backlog.

### Почему

- Шоукейс сейчас работает. Перевод — не тривиальная миграция: скрипт в
  `index.html` кастомизирует `tailwind.config` inline, некоторые кастом-цвета
  и анимации определены там. Их нужно перенести в проектный `tailwind.config.js`
  без поломки стилей.
- Приоритет — вынести шоукейс на production (ADR-005) и построить vault,
  а не вылизывать инфру.
- Шоукейс — не клиентский продукт, SEO и first-paint не критичны.

## Когда мигрировать

Триггеры для миграции (любой):
- Начнутся жалобы на медленный first paint (даже от меня)
- CDN-скрипт внезапно сломает класс/утилиту
- Будут добавляться кастомные Tailwind-плагины (jit на CDN их не умеет)
- Появится необходимость в CSS-in-JS / CSS Modules → проще переехать
  сразу в локальный билд

## Мигрируем так (когда решим)

1. Создать `tailwind.config.js` на основе inline-конфига из `index.html`
2. Создать `src/index.css` с `@tailwind base/components/utilities`
3. Импортировать `./index.css` в `main.tsx`
4. Удалить `<script src="cdn.tailwindcss.com">` и inline-конфиг
5. Проверить визуальную эквивалентность — все кастом-цвета, шрифты,
   анимации должны выглядеть идентично
6. Новый ADR «Tailwind migrated to local build» со статусом «supersedes
   ADR-004»

## Последствия

- **+** Прод работает, дедлайн не блокируется
- **–** Шоукейс остаётся тяжелее чем мог быть
- **–** Каждая новая фича, использующая Tailwind, добавляет риск тупиков
  с CDN-конфигом
