# Session Log

В конце сессии с реальной работой — запись в `docs/70 - Session Logs/`.

## Когда писать

**Писать:**
- Добавил/удалил/переименовал компоненты
- Новый ADR, новый pattern-essay, новый catalog-entry
- Изменил архитектуру / стек / деплой
- Заметный рефакторинг, перестройка папок
- Перенос компонента в другой проект (помимо reuse-entry)
- Значимое обсуждение с собой / решение развилки

**Не писать:**
- Читал код, не менял
- Починил опечатку
- Обновил зависимость без последствий

## Файл

`docs/70 - Session Logs/YYYY-MM-DD HH:mm <topic>.md`

- Дата и время — локальное, начало сессии
- `<topic>` — 2–4 слова, kebab-free (через пробел), **что сделано главное**

Примеры:
- `2026-04-21 16:00 Экстракция из library.md`
- `2026-04-21 17:30 Категоризация 215 компонентов.md`
- `2026-04-22 11:00 Vault + docs-site pipeline.md`

## Frontmatter

```yaml
---
date: 2026-04-21
topic: Экстракция из library
duration: 2h
outcomes:
  - repo_created
  - gh_pages_live
related_adr: [ADR-001]
related_components: []
related_reuse: []
---
```

## Структура

### 1. Контекст (1–2 абзаца)
Зачем взялся за это сегодня. Что было до сессии.

### 2. Что сделано
Список **в хронологическом порядке**, конкретно:
- создал репо X
- настроил Y
- решил развилку Z: выбрал A вместо B, потому что …

### 3. Развилки и решения
Если принимались архитектурные решения — каждое отдельной подсекцией с
обоснованием. Если решение тянет на ADR — создаём отдельный ADR и ссылаемся
отсюда.

### 4. Что не получилось / отложено
Честно. Баги, которые обошёл костылём, — записать. Что оставил на потом —
в `90 - Ideas & Backlog/`.

### 5. Следующие шаги
2–5 bullet-пунктов, что делать в следующей сессии. Это не план с
deadlines — это подсказка future-you.

### 6. Артефакты
Коммиты, PR-ы, внешние ссылки на деплой, скриншоты (если добавлялись в
`95 - Attachments/`).

## Стиль

- **От первого лица** или безличный — как удобнее, но единообразно в сессии.
- **Конкретика.** Не «улучшил навигацию» — «добавил prev/next в doc-page
  widget».
- **Короткие секции.** Если session log длиннее 2 экранов — сессия была
  слишком длинной, разбей на две.
- **Ссылки на файлы.** Вики-ссылки: `[[30 - Catalog/optical/ChromaCore]]`
  или пути: `src/components/optical/ChromaCore.tsx`.

## После написания

1. `docs-sync` — агент обновит Dashboard (в разделе «Последние сессии»).
2. Если возникли новые идеи — в `90 - Ideas & Backlog/`.
3. Коммит: `docs(session): 2026-04-21 — экстракция из library`.

## Anti-pattern

- **Новости-ленточка.** «Сегодня я сделал X и Y. Было весело.» — это дневник,
  не session log. Session log — инструмент для future-you, не эмоциональный
  ретроспектив.
- **Пересказ кода.** Код читается из git log. Session log — про
  **решения** и **контекст**, которых в коде нет.
