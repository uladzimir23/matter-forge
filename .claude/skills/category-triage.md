# Category Triage

16 папок в `src/components/`. Новый компонент должен попасть ровно в одну.
Этот скилл — дерево решений.

## Правило #1: категория описывает **метафору**, не технологию

Компонент может быть сделан на canvas, SVG или CSS — это не определяет
категорию. Определяет **что он изображает**.

`CausticFocus` сделан на canvas, но это *optical* — он про преломление.
`KineticArmorArray` тоже canvas, но это *kinetic* — он про движение.

## 16 категорий — семантика

| Категория | О чём | Ключевые слова |
|-----------|-------|----------------|
| **bio** | живые системы, органика, мембраны, клетки, дендриты, кожа | мембрана, мицелий, дыхание, клетка, сосуд |
| **fluid** | жидкости, капиллярность, поверхностное натяжение, осмос, вязкость | капилляр, осмос, ферро, вязкость, турбулентность |
| **optical** | свет, преломление, отражение, спектр, каустики, хроматическая дисперсия | рефракция, спектр, призма, каустика, хрома |
| **quantum** | квантовые эффекты, суперпозиция, запутанность, фаза, тахион | запутанность, суперпозиция, фаза, тахион |
| **gravity** | гравитация, орбиты, пресс, падение, притяжение | орбита, гравитация, decay, притяжение |
| **thermal** | тепло, плазма, кристаллизация, криогеника, разломы | плазма, крио, фракция, температура |
| **neural** | нейросети, синапсы, дендриты, нейронное плетение, резонанс сигнала | синапс, нейрон, резонанс, топология сети |
| **kinetic** | движение, инерция, импульс, пружины, упругость, columns | импульс, инерция, упругость, momentum, spring |
| **atmospheric** | туман, пар, воздух, дым, вапорный рендеринг | туман, пар, эфир, облачность |
| **acoustic** | звук, вибрация, haptic, tactile, резонанс, интерференция | звук, вибрация, haptic, резонанс |
| **volumetric** | объём, z-стек, слои-3D, голография, параллакс глубины | голограмма, z-stack, объём, параллакс |
| **structural** | геометрия, решётки, стекинг, пульсация конструкции, adaptive geometry | решётка, стекинг, геометрия |
| **topology** | топология, морфинг форм, струнная геометрия, spatial switcher | морф, топология, струны |
| **state** | состояния, переходы состояний, протоколы, энтропия, временные последовательности | state, morpher, протокол, энтропия |
| **molecular** | молекулярный синтез, сборка, атомарный дизайн, компонентная анатомия | молекула, атом, синтез, blueprint |
| **system** | **escape hatch.** Служебные виджеты (CommandCenter, DevOpsPipeline, SyntaxIDE) — не визуальная метафора, а инструмент шоукейса | — |

## Дерево решений

```
Это визуальная метафора или служебный виджет шоукейса?
├── служебный (консоль, терминал, IDE, pipeline) → system
└── визуальная метафора →
    Что он имитирует?
    ├── живое (клетки, дыхание, кожа) → bio
    ├── жидкость → fluid
    ├── свет / преломление / спектр → optical
    ├── звук / вибрацию / тактильность → acoustic
    ├── квантовые эффекты → quantum
    ├── гравитацию / орбиты → gravity
    ├── температуру / плазму → thermal
    ├── нейросетевую топологию → neural
    ├── механическое движение / инерцию → kinetic
    ├── газ / туман / пар → atmospheric
    ├── объём / z-слои / голограмму → volumetric
    ├── геометрическую структуру (решётка, стекинг) → structural
    ├── топологические морфинги (форма→форма) → topology
    ├── состояние и его переходы → state
    └── молекулярную/атомарную сборку → molecular
```

## Пограничные случаи

- **«Компонент и про движение, и про жидкость»** (напр. VortexFluidTrail) —
  приоритет по первичной метафоре. Trail движется, но **изображает** вихрь
  жидкости → `fluid`. Вторичная метафора попадает в теги.
- **«Компонент и про звук, и про haptic»** — оба в `acoustic`. Акустика
  и haptic идут рядом (оба про вибрации разной частоты).
- **«Это просто красивая сетка без метафоры»** — `structural`.
- **«Это иконка-морф»** (IconMorphSequence) — это `topology`, потому что
  про морфинг форм.
- **«Это голографический объёмный чарт»** — `volumetric` (объём важнее,
  чем chart).
- **«Это HUD поверх интерфейса»** — смотрим на эффект: если преломляет —
  `optical`, если про нейро-паттерны — `neural`, если просто служебный —
  `system`.

## Если всё ещё не уверен

Проверь какой глагол описывает компонент:
- *дышит, растёт, делится* → bio
- *течёт, капает, вязнет* → fluid
- *преломляет, отражает, рассеивает* → optical
- *вибрирует, пульсирует, звенит* → acoustic
- *запутан, суперпозиционирует* → quantum
- *притягивает, вращается по орбите* → gravity
- *нагревается, остывает, кристаллизуется* → thermal
- *передаёт сигнал по сети, резонирует между узлами* → neural
- *движется, отскакивает, затухает* → kinetic
- *испаряется, окутывает, рассеивается в воздухе* → atmospheric
- *проецируется в объёме, даёт глубину* → volumetric
- *удерживает структуру, решётит* → structural
- *морфит форму, перестраивает топологию* → topology
- *меняет состояние, переходит между режимами* → state
- *синтезируется из атомов, собирается* → molecular
- *управляет, логирует, диагностирует шоукейс* → system

## После triage

1. Файл в `src/components/<cat>/Name.tsx`
2. Catalog-entry в `docs/30 - Catalog/<cat>/Name.mdx` (см. `catalog-entry` скилл)
3. Если компонент претендует на мета-паттерн — посмотреть, попадает ли он
   в существующее эссе в `50 - Patterns & Motifs/`, добавить ссылку.

## Ре-категоризация

Если через месяц стало понятно, что компонент живёт в неправильной папке:
1. `git mv src/components/<old>/<Name>.tsx src/components/<new>/<Name>.tsx`
2. Rewrite импортов (см. `scripts/mf-recategorize.py` если есть, иначе вручную)
3. Обновить `category:` во frontmatter catalog-entry
4. Обновить ссылки в `50 - Patterns & Motifs/` и `60 - Reuse Log/`
5. Session log
